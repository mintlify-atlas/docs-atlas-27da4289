## Installation

<Steps>
  <Step title="Clone and install dependencies">
    First, clone the repository and install the required packages:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>

    The API uses the following key dependencies:
    - `express` - Web framework
    - `express-jwt` & `jwks-rsa` - Auth0 authentication
    - `multer` - File upload handling
    - `knex` & `pg` - PostgreSQL database integration
  </Step>

  <Step title="Set up your environment variables">
    Create a `.env` file in the root directory with the following variables:

    ```bash .env
    # Server Configuration
    PORT=3000

    # PostgreSQL Database
    PGHOST=your_database_host
    PGPORT=5432
    PGUSER=your_database_user
    PGPASSWORD=your_database_password
    PGDATABASE=cat_data

    # Auth0 Configuration
    AUTH0_DOMAIN=https://your-domain.auth0.com/
    AUTH0_AUDIENCE=your_api_identifier
    ```

    <Warning>
    Make sure `AUTH0_DOMAIN` ends with a trailing slash (`/`) as the middleware appends `.well-known/jwks.json` directly to it.
    </Warning>
  </Step>

  <Step title="Set up the database">
    Run the Knex migrations to create the necessary database tables:

    ```bash
    npx knex migrate:latest
    ```

    This creates the `images` table required for storing cat image metadata.
  </Step>

  <Step title="Start the development server">
    Run the development server with hot-reload enabled:

    ```bash
    npm run dev
    ```

    The API will start on `http://localhost:3000` (or your configured PORT).
  </Step>
</Steps>

## Making Your First Request

### Upload a Cat Image

Upload a cat image to the API using multipart/form-data:

```bash
curl -X POST http://localhost:3000/api/upload \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -F "file=@/path/to/cat-image.jpg"
```

<Note>
Only JPG and PNG image formats are accepted. Other file types will be rejected with an error.
</Note>

**Response:**

```json
{
  "id": 1,
  "filename": "a1b2c3d4-5678-90ab-cdef-1234567890ab-cat-image.jpg"
}
```

The API automatically generates a UUID-prefixed filename to prevent naming conflicts.

### Retrieve All Images

Get a list of all uploaded cat images:

```bash
curl http://localhost:3000/api/images \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

**Response:**

```json
[
  {
    "id": 1,
    "name": "a1b2c3d4-5678-90ab-cdef-1234567890ab-cat-image.jpg"
  },
  {
    "id": 2,
    "name": "b2c3d4e5-6789-01bc-def0-234567890abc-another-cat.png"
  }
]
```

### Get a Specific Image

Retrieve a specific cat image by its ID:

```bash
curl http://localhost:3000/api/image/1 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  --output cat.jpg
```

This endpoint returns the actual image file for display or download.

### Delete an Image

Remove an image from both the filesystem and database:

```bash
curl -X DELETE http://localhost:3000/api/image/1 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

**Response:** `204 No Content`

<Warning>
Deletion is permanent and removes both the file from the `images/` directory and the database record.
</Warning>

## Complete Example Workflow

Here's a complete example of uploading and retrieving a cat image:

```bash
# 1. Upload a cat image
RESPONSE=$(curl -s -X POST http://localhost:3000/api/upload \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -F "file=@cat.jpg")

# 2. Extract the image ID from the response
IMAGE_ID=$(echo $RESPONSE | jq -r '.id')

# 3. Retrieve the image
curl http://localhost:3000/api/image/$IMAGE_ID \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  --output retrieved-cat.jpg

# 4. List all images to verify
curl http://localhost:3000/api/images \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

<Tip>
All endpoints require authentication. See the [Authentication](/authentication) page to learn how to obtain a JWT token from Auth0.
</Tip>
