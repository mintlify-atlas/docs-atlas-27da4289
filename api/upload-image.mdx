## POST /api/upload

Uploads a new cat image file to the server and stores metadata in the database.

<Note>
Implementation: src/index.ts:42-57
</Note>

### Authentication

Requires JWT authentication via the `checkJwt` middleware.

### Request Parameters

<ParamField body="file" type="file" required>
  The image file to upload. Must be sent as `multipart/form-data` with the field name `file`.
</ParamField>

### File Validation

The API enforces strict file type validation:

<Warning>
Only JPG and PNG files are allowed. Other file types will be rejected with an error.
</Warning>

**Allowed MIME types:**
- `image/jpeg`
- `image/png`

**File naming:**
Uploaded files are automatically renamed using the pattern `{uuid}-{original-filename}` to prevent conflicts (src/index.ts:37).

### Response

<ResponseField name="id" type="number">
  The database ID of the uploaded image record.
</ResponseField>

<ResponseField name="filename" type="string">
  The generated filename stored on the server (includes UUID prefix).
</ResponseField>

### Response Codes

| Status Code | Description |
|-------------|-------------|
| `201` | Image uploaded successfully |
| `400` | Missing file or invalid file type |
| `401` | Missing or invalid JWT token |

### Example Request

```bash
curl -X POST http://localhost:3000/api/upload \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -F "file=@/path/to/cat-image.jpg"
```

### Example Response

```json
{
  "id": 42,
  "filename": "a1b2c3d4-e5f6-4789-a012-3456789abcde-fluffy-cat.jpg"
}
```

### Error Response

```json
{
  "error": "Missing file to upload"
}
```

or

```json
{
  "error": "Only JPG and PNG files are allowed."
}
```

## Implementation Details

### Multer Configuration

The upload endpoint uses multer with custom storage and file filtering (src/index.ts:27-40):

```typescript
const fileFilter: multer.Options['fileFilter'] = (req, file, cb) => {
  const allowedTypes = ['image/jpeg', 'image/png'];
  if (!allowedTypes.includes(file.mimetype)) {
    return cb(new Error('Only JPG and PNG files are allowed.'));
  }
  cb(null, true);
};

const storage = multer.diskStorage({
  destination: (req, file, cb) => cb(null, imagesDir),
  filename: (req, file, cb) => cb(null, uuidv4() + '-' + file.originalname),
});

const upload = multer({ storage, fileFilter });
```

### Database Storage

After successful file upload, the filename is stored in the database using the `addImage` function from db-functions.ts:3-11:

```typescript
const id = await addImage(req.file.filename);
```
